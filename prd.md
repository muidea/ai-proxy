# 项目文档：轻量级 LLM 本地代理网关 (Golang)

## 1. 项目背景
在本地使用各种 AI 客户端（如 NextChat, LobeChat, ChatBox 等）时，通常需要直接连接多个云端模型 API。本项目旨在构建一个运行在本地的轻量级代理服务（Gateway），统一管理 API 调用，并实时统计不同模型的 Token 消耗情况，以便用户直观掌握使用成本。

---

## 2. 项目目标 (Goals)

### 2.1 核心功能目标
*   **统一协议转换**：作为本地网关，接收客户端请求，并透明转发至目标供应商（OpenAI, DeepSeek, Anthropic 等）。
*   **Token 自动审计**：
    *   针对**非流式响应**，解析返回包中的 `usage` 字段。
    *   针对**流式响应 (Streaming)**，实时拦截数据流，解析流结束前的 Token 统计信息。
*   **模型消耗透明化**：在控制台/终端实时打印当前请求的模型、耗时及 Token 消耗。

### 2.2 易用性与部署目标
*   **零环境依赖**：采用 Golang 开发，编译为单二进制文件，无需安装数据库或中间件。
*   **极简持久化**：不使用复杂数据库，将调用流水以追加模式写入本地文本文件（如 `stats.csv`）。
*   **低侵入性**：兼容标准 OpenAI 协议，本地应用只需修改 `API_BASE_URL` 即可接入。

### 2.3 性能目标
*   **极低延迟**：代理层引入的额外延迟（TTFT）应小于 20ms，确保不影响对话体验。
*   **轻量占用**：静态内存占用控制在 30MB 以内。

---

## 3. 完成定义 (Definition of Done, DoD)

### 3.1 功能准则 (Functional)
- [ ] **基础转发**：能够正确代理请求并获得预期回复。
- [ ] **流式支持**：成功处理 `stream: true` 请求，客户端能够流畅看到字符逐个弹出。
- [ ] **Token 捕获**：
    - [ ] 对于返回 `usage` 字段的模型，能够准确提取 `prompt_tokens` 和 `completion_tokens`。
    - [ ] 对于不提供流式 usage 的场景，具备简单的字符数估算逻辑（或使用 `tiktoken-go` 库计算）。
- [ ] **异常透传**：当上游 API Key 失效或余额不足时，能将原始错误状态码（如 401, 429）透传给客户端。

### 3.2 存储与记录准则 (Data & Logging)
- [ ] **本地流水账**：每次调用结束后，自动在当前目录的 `usage.csv` 中追加一行记录，包含：`时间, 模型, 输入Token, 输出Token, 总耗时`。
- [ ] **控制台反馈**：在终端通过颜色区分或表格形式，实时输出当次调用的摘要。

### 3.3 技术准则 (Technical)
- [ ] **并发安全**：支持本地多个 AI 客户端同时发起请求而不会导致数据错乱或进程崩溃。
- [ ] **配置管理**：支持通过 `config.yaml`配置 API Keys 及代理端口。
- [ ] **单文件交付**：提供可执行文件，无需在目标机器配置 Go 运行环境。
